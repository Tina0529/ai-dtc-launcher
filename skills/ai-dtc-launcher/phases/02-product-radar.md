# Phase 2: 选品雷达 (Product Radar)

## 阶段目标

在 Phase 1 验证通过的品类市场中，通过爆款内容分析和供应链调研，找到具体要销售的产品。从"品类"细化到"产品"，确定 1-3 个利润可行的候选产品。

---

## 输入

| 参数 | 来源 | 说明 |
|------|------|------|
| 验证通过的品类 | Phase 1b 输出 | 品类名称及市场验证报告 |
| 竞品价格区间 | Phase 1b 输出 | 头部玩家的定价数据 |
| 供应链信息 | Phase 1b 输出 | 1688 供应商/MOQ/价格 |
| 目标市场 | meta.json | china / japan / global |

---

## 执行步骤

### Step 1: 爆款内容采集

在主流内容平台搜索该品类的热门内容，识别消费者正在关注什么具体产品。

#### 1.1 搜索关键词准备

```
品类关键词: {category_name}
扩展关键词:
  - "{category_name} 推荐"
  - "{category_name} 测评"
  - "{category_name} 好物分享"
  - "{category_name} 平价好物"
  - "{category_name} 开箱"
  - "好用的{category_name}"
```

#### 1.2 平台内容采集

根据数据采集模式分别执行：

**Auto 模式（B站）**：
- 使用 WebSearch 搜索 B站 相关视频
- 筛选条件：播放量 > 1 万，近 6 个月内发布
- 采集 Top 20 内容的标题、简介、播放量、弹幕关键词

**Semi-auto 模式（抖音/小红书）**：
```
提示用户：
"请在以下平台搜索 '{category_name}'，将以下信息粘贴给我：

【抖音】搜索 '{category_name}'，筛选'最多点赞'，复制前 20 个视频的：
- 标题
- 点赞数
- 评论数
- 视频描述/字幕（如有）

【小红书】搜索 '{category_name}'，筛选'最热'，复制前 20 篇笔记的：
- 标题
- 点赞/收藏/评论数
- 正文内容

或者您也可以直接截图给我，我来识别提取。"
```

**Third-party 模式（可选）**：
```
提示用户：
"如果您有蝉妈妈/飞瓜数据的访问权限，可以导出以下数据的 CSV 文件：
- '{category_name}' 品类近 30 天热门商品排行
- '{category_name}' 相关视频带货排行
导出后上传给我，我来分析。"
```

#### 1.3 高互动内容筛选

从采集的内容中筛选高互动内容：

| 筛选标准 | 抖音 | 小红书 | B站 |
|---------|------|--------|-----|
| 点赞门槛 | > 1 万 | > 5000 | > 3000 |
| 评论门槛 | > 500 | > 200 | > 100 |
| 时间范围 | 近 3 个月 | 近 6 个月 | 近 6 个月 |

保留符合条件的 Top 20 条内容进入下一步。

#### 1.4 评论采集

对 Top 20 条内容，采集用户评论中与产品相关的关键信息：
- 用户提到了哪些具体产品/品牌
- 用户的购买意向和偏好
- 用户的痛点和需求表达

---

### Step 2: AI 商品提取

对采集到的内容和评论进行 AI 分析，提取具体商品。

**内置 Prompt**：

```
请从以下内容中提取所有被提及或展示的商品名称和品牌名称。

要求：
1. 提取具体的产品名（不是品类名），例如"小熊便携榨汁杯 LLJ-D04"而不是"便携榨汁杯"
2. 如果内容中只提到品类没有具体产品，标记为"品类提及"
3. 按提及频率从高到低排序
4. 对相似产品进行聚类合并（如"九阳榨汁杯"和"九阳便携果汁杯"合并）
5. 标注每个产品被提及的场景（推荐/测评/吐槽/比较）

内容数据：
{content_data}
```

**输出格式**：

| 排名 | 产品名/品牌 | 提及次数 | 提及场景 | 平均评价倾向 | 价格带 |
|------|-----------|---------|---------|------------|--------|
| 1 | {product_name} | {count} | {推荐/测评/...} | {正面/中性/负面} | {price} |
| 2 | ... | ... | ... | ... | ... |

**聚类统计**：
```
提及频率 Top 5 产品类型：
1. {product_type_1} — 被提及 {N} 次（占比 {pct}%）
2. {product_type_2} — 被提及 {N} 次（占比 {pct}%）
...
```

---

### Step 3: 趋势交叉验证

对提取出的 Top 5-10 个具体产品/产品类型，执行趋势验证。

**操作**：加载 `engines/trend-crosscheck.md`，对每个产品执行验证。

**与 Phase 1a 的区别**：这里验证的是具体产品而非品类，关键词更精确。

**快速验证模式**（节约时间）：
- 每个产品至少搜索 3 个数据源（中国 2 + 全球 1）
- 置信度 MEDIUM 以上即可通过
- 重点关注是否有下降趋势（淘汰信号）

---

### Step 4: 利润可行性速判

对通过趋势验证的产品，进行利润快速测算。

#### 4.1 采购价查询

使用 `ecommerce-searcher` 在 1688 搜索每个候选产品：

```
搜索参数：
- 关键词: {product_name}
- 平台: 1688
- 排序: 综合排序 / 销量排序
```

**记录数据**：

| 产品 | 1688 最低价 | 1688 均价 | MOQ | 可定制 | 供应商评级 |
|------|----------|---------|-----|--------|-----------|
| {product_1} | {min_price} | {avg_price} | {moq} | {是/否} | {rating} |
| ... | ... | ... | ... | ... | ... |

#### 4.2 竞品售价查询

使用 WebSearch 查询该产品在各电商平台的零售价：

```
搜索查询：
- "淘宝 {product_name} 价格"
- "拼多多 {product_name} 价格"
- "抖音小店 {product_name} 价格"
```

**记录数据**：

| 产品 | 淘宝均价 | 拼多多均价 | 抖音均价 | 品牌溢价空间 |
|------|---------|----------|---------|------------|
| {product_1} | {price} | {price} | {price} | {有/有限/无} |
| ... | ... | ... | ... | ... |

#### 4.3 毛利率计算

```
采购成本 = 1688 均价
预估售价 = 竞品均价 × 0.8 ~ 1.2（根据定位调整）
预估毛利率 = (预估售价 - 采购成本 - 物流包装约5元 - 平台费率约5%) / 预估售价

毛利率判定：
  ≥ 40% → 🟢 绿灯：利润空间充足
  20-40% → 🟡 黄灯：利润偏薄，需要规模或差异化
  < 20% → 🔴 红灯：利润不足
```

---

### Step 5: 爆品潜力评分

**操作**：加载 `engines/hit-product-scorer.md`，对每个候选产品进行打分。

> 如果 `engines/hit-product-scorer.md` 尚未创建，使用以下简化评分：

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 内容热度 | 25% | 相关爆款内容数量和互动量 |
| 趋势方向 | 25% | 上升趋势得高分 |
| 利润空间 | 25% | 毛利率 40%+ 得满分 |
| 差异化空间 | 15% | 现有产品是否有明显可改进空间 |
| 供应链友好度 | 10% | MOQ 低、供应商多、可定制 |

## 检查点

**暂停执行，展示排行榜给用户，等待用户选择。**

提示用户：
```
以上是 "{品类名}" 品类中的候选产品排行榜。

请选择 1-3 个您想做的产品。选择建议：
- 优先选择利润绿灯 + 趋势上升的产品
- 初次创业建议只选 1 个产品，聚焦做好

您也可以：
- 要求我对某个产品做更深入分析
- 提出您自己发现的产品让我评估
- 调整价格策略后重新计算利润
```

**用户选择后**：
- 更新 `meta.json`：记录选定产品，`current_phase` 设为 `phase-3`
- 保存报告到 `product-candidates.md`

## 失败路径

### 情况 1: 所有产品利润为红灯

```
分析原因：
1. 采购价太高 → 搜索更多供应商/考虑直接联系工厂
2. 竞品售价太低 → 品类可能已陷入价格战，考虑差异化提价
3. 品类天然低毛利 → 需要靠规模或高频复购
↓
优化建议：
- 在 1688 搜索 "源头工厂" + {品类名}
- 考虑添加定制/差异化元素以支撑更高售价
- 考虑组合销售/套装策略提升客单价
↓
优化后仍全红灯 → 返回 Phase 1b，重新评估品类或选择新品类
```

### 情况 2: 爆款内容采集数据不足

```
可能原因：品类太小众 / 平台数据获取困难
↓
行动：
- 扩大搜索关键词范围（加入近义词、上级品类）
- 增加搜索平台（加入微博、知乎等）
- 降低互动量门槛
↓
仍然不足 → 提示用户该品类可能缺乏足够的内容生态支撑
  建议："该品类的社交内容量较少，可能意味着：
        1. 市场尚处于非常早期 — 内容红利大但需要教育市场
        2. 产品不适合内容营销 — 需要其他获客方式
        您想继续（以教育市场的方式）还是换品类？"
```

### 情况 3: 用户提出的产品不在列表中

```
行动：
- 对用户指定的产品执行 Step 3-5 的验证流程
- 补充到排行榜中
- 如果评分较高，尊重用户判断
- 如果评分偏低，如实告知风险但不阻止
```

---

## 注意事项

1. 产品粒度要适中：不要太泛（如"杯子"），也不要太细（如"350ml 双层玻璃杯薄荷绿磨砂款"）
2. 利润测算中，物流包装费用按 5 元/件估算，后续 Phase 5 会精确计算
3. 平台费率按 5% 统一估算，不同平台实际费率有差异
4. 采购价使用 1688 均价而非最低价，最低价往往质量不可靠
5. Semi-auto 模式下，引导用户提供尽可能结构化的数据，减少 AI 解析误差
6. 如果用户能提供蝉妈妈/飞瓜等工具的数据，分析质量会显著提升
