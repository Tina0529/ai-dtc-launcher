# 三秒病毒法则 (3-Second Virus Rule)

## 用途

短视频的前 3 秒决定了用户是否继续观看。本引擎提供一套可执行的"钩子设计"方法论，确保每条视频的开头都能在 3 秒内抓住观众注意力。通过 4 种钩子类型的组合运用和 AI 批量生成，提高完播率和传播效率。

---

## 核心规则

> **前 3 秒必须包含 ≥ 2 种钩子类型的组合。** 单一钩子的留存效果远不如组合钩子。

---

## 4 种钩子类型

### 类型 1: 视觉冲击钩 (Visual Hook)

**定义**：通过画面的强烈视觉刺激，在第一帧就锁住眼球。利用人类大脑对"异常画面"的本能关注。

**3 种子类型**：

| 子类型 | 技法 | 示例 |
|--------|------|------|
| **反差** | 前后对比极端化，视觉落差越大越好 | 油腻厨房 → 一擦即净的闪亮灶台（清洁产品） |
| **意外** | 出现不符合预期的画面，制造认知冲突 | 把口红涂在手背上用水冲 → 纹丝不动（持妆力演示） |
| **高颜值产品特写** | 极致的产品微距/慢动作/光影展示 | 护肤精华滴落的慢动作特写，液滴在光线下折射 |

**执行要点**：
- 第 1 帧必须是最有冲击力的画面，不要"慢慢来"
- 使用近景或特写，避免远景开场
- 画面亮度要足够（暗画面完播率低 30%+）

### 类型 2: 痛点共鸣钩 (Pain Point Hook)

**定义**：用一句话精准戳中目标用户的日常困扰，让用户产生"说的就是我"的共鸣。

**句式模板**：

| 句式 | 示例 |
|------|------|
| "你是不是也遇到过……" | "你是不是也遇到过，洗完脸 2 小时就开始出油？" |
| "每次……都让我崩溃" | "每次做饭后清理油烟机都让我崩溃" |
| "为什么……总是……" | "为什么花了几百块的面霜，皮肤还是干到起皮？" |
| "最怕……的时候……" | "最怕约会的时候，口红沾杯沾得到处都是" |
| "{身份}最大的烦恼就是……" | "养猫人最大的烦恼就是——家里永远有猫毛" |

**执行要点**：
- 痛点必须来自 brand-vacuum-detector 提取的高频痛点 Top 5
- 使用口语化表达，不要书面语
- 语气自然，像朋友聊天，不要像广告播报

### 类型 3: 悬念好奇钩 (Curiosity Hook)

**定义**：制造信息缺口，利用人类"必须知道答案"的好奇心驱动继续观看。

**句式模板**：

| 句式 | 示例 |
|------|------|
| "99% 的人不知道……" | "99% 的人不知道，卸妆油还能这样用" |
| "千万别……否则……" | "千万别在早上敷面膜，否则你的皮肤会……" |
| "我花了 X 年才发现……" | "我花了 3 年才发现，收纳的核心不是买收纳盒" |
| "这个方法 {权威} 都在用" | "这个方法皮肤科医生都在用，但很少有人知道" |
| "看到最后有惊喜" | "坚持看到最后，我会告诉你在哪里买到正品" |

**执行要点**：
- 悬念必须在视频中有回应（不要标题党无落地）
- 数字越具体越有吸引力（"3 个技巧"比"几个技巧"好）
- 权威背书要真实可查

### 类型 4: 利益承诺钩 (Benefit Hook)

**定义**：直接告诉用户"看这条视频你能得到什么"，用具体利益驱动观看。

**句式模板**：

| 句式 | 示例 |
|------|------|
| "省了我 XX 块钱" | "这个平替省了我 2000 块钱，效果一模一样" |
| "用了它之后，我再也不……" | "用了它之后，我再也不用担心脱妆了" |
| "X 天见效/X 秒搞定" | "7 天肉眼可见的变化，不是广告是亲测" |
| "后悔没早点知道" | "这个收纳神器后悔没早点知道，直接省了一半空间" |
| "闺蜜/同事都问我……" | "同事都问我用了什么，皮肤怎么突然变好了" |

**执行要点**：
- 利益必须具体、可量化（有数字比没数字好）
- 承诺必须真实可兑现，避免过度夸大导致差评
- 用第一人称增强真实感

## 钩子组合策略

### 推荐组合（按效果排序）

| 优先级 | 组合 | 适用场景 | 示例 |
|--------|------|---------|------|
| ★★★★★ | 视觉冲击 + 痛点共鸣 | 功能型产品展示 | 【画面】油腻灶台特写 →【旁白】"你是不是也受够了厨房油污？" |
| ★★★★★ | 悬念好奇 + 视觉冲击 | 新奇/创新产品 | 【旁白】"99%的人不知道这个用法" →【画面】产品意外用途演示 |
| ★★★★☆ | 痛点共鸣 + 利益承诺 | 解决方案型内容 | 【旁白】"每次卸妆都搓到脸红？这瓶精华让我告别卸妆棉" |
| ★★★★☆ | 利益承诺 + 视觉冲击 | 对比/测评型内容 | 【旁白】"省了我3000块" →【画面】平替 vs 大牌的对比特写 |
| ★★★☆☆ | 悬念好奇 + 利益承诺 | 教程/攻略型内容 | 【旁白】"皮肤科医生不会告诉你，这个方法能省一半护肤费" |
| ★★★☆☆ | 痛点共鸣 + 悬念好奇 | 知识型内容 | 【旁白】"为什么你的面膜总是没效果？真相你可能不敢信" |

### 禁忌组合

- 不要将 4 种钩子全部塞进前 3 秒，信息过载反而降低留存
- 同类型钩子不要重复（如两个悬念句式连用会显得刻意）

## AI 批量生成开头方案

使用以下 prompt 批量生成 5 个视频开头方案：

```
你是一个短视频创意总监，擅长设计高完播率的视频开头。请为以下产品生成 5 个视频开头方案。

**产品信息**:
- 产品名称: {product_name}
- 品类: {category}
- 核心卖点: {key_selling_point}
- 目标人群: {target_persona}（来自 persona-brand-matcher）
- 高频痛点: {top_3_pain_points}（来自 brand-vacuum-detector）

**要求**:
1. 每个方案必须在前 3 秒组合使用 ≥ 2 种钩子类型
2. 5 个方案分别使用不同的钩子组合
3. 每个方案包含：
   - 钩子组合类型标注
   - 前 3 秒的画面脚本（逐秒描述）
   - 前 3 秒的旁白/文字脚本
   - 预估的吸引力评分（1-10）和理由

**输出格式**:

### 方案 1: {方案名}
- **钩子组合**: {类型A} + {类型B}
- **画面脚本**:
  - 第 0-1 秒: {画面描述}
  - 第 1-2 秒: {画面描述}
  - 第 2-3 秒: {画面描述}
- **旁白/字幕**: "{文字内容}"
- **吸引力评分**: {X}/10
- **评分理由**: {为什么这个开头能留住观众}

### 方案 2: ...
（以此类推，共 5 个方案）

### 推荐排序

按吸引力评分从高到低排序，推荐优先拍摄 Top 2 方案。
```

## 与 Content Scoring 集成

### 完播率预测

视频发布后，将前 3 秒钩子方案与实际完播率数据关联，建立反馈循环：

| 指标 | 优秀 | 合格 | 不合格 |
|------|------|------|--------|
| 3 秒留存率 | > 70% | 50-70% | < 50% |
| 完播率（15s 视频） | > 40% | 25-40% | < 25% |
| 完播率（30s 视频） | > 30% | 15-30% | < 15% |
| 完播率（60s 视频） | > 20% | 10-20% | < 10% |

### 评判标准

对每个已发布视频，按以下标准评判前 3 秒钩子的效果：

```
IF 3秒留存率 ≥ 70%:
    → 钩子有效，记录该组合为"已验证有效组合"
    → 后续同品类视频优先复用此组合

IF 3秒留存率 50-70%:
    → 钩子基本有效但有优化空间
    → 分析：是画面不够冲击？还是文案不够痛？
    → 生成优化版方案后 A/B 测试

IF 3秒留存率 < 50%:
    → 钩子失败
    → 弃用该组合，切换到其他组合重新拍摄
```

## 优化迭代流程

```
┌─────────────────────────────────────────────────┐
│ 1. AI 生成 5 个开头方案                           │
│    ↓                                             │
│ 2. 选择 Top 2 方案拍摄并发布                      │
│    ↓                                             │
│ 3. 观察 24-48 小时数据（3秒留存率 + 完播率）       │
│    ↓                                             │
│ 4. 评判结果                                      │
│    ├── 优秀 → 加大投放（→ precision-funnel）      │
│    ├── 合格 → 微调优化后重新发布                   │
│    └── 不合格 → 弃用，选择剩余方案中的 Top 1 重拍  │
│    ↓                                             │
│ 5. 积累有效组合数据库，持续优化生成质量             │
└─────────────────────────────────────────────────┘
```

## 注意事项

1. 前 3 秒文案不要超过 20 个字，太长来不及读完。
2. 画面切换速度在前 3 秒应 ≥ 2 次镜头变化，静态画面留存率低。
3. 背景音乐在前 3 秒要有节奏感强的"hook point"，配合画面冲击。
4. 竖屏视频（9:16）的产品/人物应占据画面 60%+ 面积，不要留太多空白。
5. 字幕要大、要醒目，位于画面中上部（避开底部操作区域）。
