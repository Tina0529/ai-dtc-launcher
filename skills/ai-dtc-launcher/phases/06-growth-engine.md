# Phase 6: 增长引擎 (Growth Engine)

## 阶段目标

通过存量粉丝激活、爆款内容生产和精准投放三大引擎驱动产品销售。核心策略：先用存量粉丝验证 MVP，达标后再投入公域投放，实现"低风险验证 → 高效放大"的增长路径。

---

## 输入

| 参数 | 来源 | 说明 |
|------|------|------|
| 品牌简报 | Phase 4 输出 | 品牌视觉/调性/目标人群 |
| 产品方案包 | Phase 5 输出 | 产品信息/素材/定价 |
| 消费者洞察 | Phase 3 输出 | 痛点/需求/话术素材 |
| Launch Checklist | Phase 5 输出 | 确认所有准备就绪 |
| 各平台粉丝数 | 用户信息 | 用户现有粉丝基础（需在 Step 1 开头确认） |
| 目标市场 | meta.json | china / japan / global |

## 执行步骤

### Step 1: 存量激活 MVP 验证（核心差异化策略）

**这是整个增长阶段最关键的一步**。利用现有粉丝基础作为"免费流量池"，验证产品是否真正有市场需求。

#### 1.0 粉丝基础确认

**在开始 MVP 验证前，先确认用户在各平台的粉丝基础。**

```
提示用户：
"在开始增长策略之前，请告诉我您在各平台的粉丝情况：

- 视频号：___ 粉丝
- 抖音：___ 粉丝
- 小红书：___ 粉丝
- B站：___ 粉丝
- 其他平台：___

这将决定我们采用哪种启动策略。"
```

**路径判定**：

| 粉丝情况 | 判定 | 策略 |
|---------|------|------|
| 任一平台 ≥ 1000 粉丝 | 有存量粉丝 | 执行存量激活策略（Step 1.1 → Step 1.5） |
| 所有平台 < 1000 粉丝 | 无存量粉丝 | 执行冷启动策略（见下方） |

**冷启动策略（无存量粉丝时）**：
```
如果用户在所有平台粉丝均 < 1000，采用以下替代路径：

1. 跳过 Step 1，先执行 Step 2（爆款内容生产）
   - 用高质量内容积累首批粉丝
   - 目标：30 天内在主力平台积累 ≥ 1000 粉丝

2. 同步执行低成本 MVP 验证：
   - 在闲鱼/转转发布产品测试市场反应
   - 在微信朋友圈/社群小范围测试
   - 找 3-5 个 KOC 合作（纯佣金模式，零成本）

3. 粉丝达到 1000 后，回到 Step 1 执行存量激活策略

冷启动期预期时间：2-4 周
```


#### 1.3 MVP 成功门槛

**按品类客单价分级**（纯自然流量，不投广告）：

| 客单价区间 | 7天订单数目标 | 或 7天GMV目标 | 预警线 |
|-----------|-------------|-------------|--------|
| 低客单价（<50元） | ≥ 50 单 | ≥ 1,500 元 | < 25 单 |
| 中客单价（50-300元） | ≥ 30 单 | ≥ 3,000 元 | < 15 单 |
| 高客单价（>300元） | ≥ 10 单 | ≥ 3,000 元 | < 5 单 |

**通用指标**（所有客单价均适用）：

| 指标 | 目标值 | 计算方式 | 预警线 |
|------|--------|---------|--------|
| 退货率 | < 15% | 退货数 / 订单数 | > 20% |
| 好评率 | > 80% | 好评数 / 评价总数 | < 60% |
| 复询率 | > 5% | 重复咨询/购买用户 / 总用户 | < 2% |

> 注：订单数和 GMV 满足其一即视为达标。例如高客单价产品 7 天卖 8 单但 GMV 达到 4000 元，视为达标。

#### 1.4 MVP 追踪

每日记录 MVP 关键指标，使用 `templates/daily-tracker.md` 的 MVP 追踪部分。

#### 1.5 MVP 判定

```
7 天后判定：

✅ 全部达标 → 进入 Step 2 公域投放
  "恭喜！MVP 验证通过，产品有真实的市场需求。
   接下来我们进入公域爆款内容生产和精准投放。"

⚠️ 部分达标 → 诊断问题，调整后延长测试
  - 订单不足但退货率低 → 流量问题 → 优化视频内容/发布时间
  - 订单达标但退货率高 → 产品/期望差问题 → 优化详情页/产品
  - 好评率低 → 产品品质问题 → 联系供应商改进
  延长 7 天观察

❌ 全部未达标 → 暂停，深入分析
  第 1 次失败 → 诊断根因 → 调整策略 → 重试
  第 2 次失败 → 暂停推进
  "MVP 两次未达标，建议暂停并进行根因分析：
   1. 产品是否真的解决了消费者痛点？
   2. 定价是否合理？
   3. 目标人群是否匹配？
   4. 是否需要返回 Phase 2/3 重新评估？"
```

**检查点：MVP 达标后，用户确认进入公域投放**

### Step 2: 爆款内容生产

#### 2.0 大促节点日历

**中国电商大促日历**（提前 2-4 周准备）：

| 月份 | 大促节点 | 平台 | 建议动作 |
|------|---------|------|---------|
| 1月 | 年货节 | 全平台 | 年货/礼品类产品重点发力 |
| 3月 | 女神节（3.8） | 全平台 | 女性消费品重点发力 |
| 6月 | 618 大促 | 全平台 | 年中最大促销节点 |
| 8月 | 818 发烧节 | 苏宁/抖音 | 电子数码类重点 |
| 11月 | 双11 | 全平台 | 全年最大促销节点 |
| 12月 | 双12 | 全平台 | 年终清仓/圣诞礼品 |

**大促利用策略**：
```
1. 产品上线时机：尽量提前大促 2-4 周上线
   - 大促前的流量竞争较小，获客成本低
   - 可以用低价流量做 MVP 验证
   - 积累一定销量和评价后，大促期间更有竞争力

2. 大促窗口期参与：
   - 如果恰好赶上大促窗口，积极报名平台活动
   - 视频号小店/抖音小店均有大促专属活动入口
   - 新店铺通常有流量扶持，大促期间扶持力度更大

3. 大促后复盘：
   - 大促期间的数据（转化率、退货率）是重要参考
   - 注意区分"大促冲动消费"和"真实需求"
   - 大促后退货率通常偏高，属正常现象
```

### Step 4: 效果追踪与迭代

#### 4.1 每日数据追踪

使用 `templates/daily-tracker.md` 每日记录所有数据。

```
每日追踪流程：
1. 收集各平台数据
2. 填写 daily-tracker.md
3. AI 自动分析趋势和异常
4. 生成当日关键发现和明日行动计划
```

#### 4.2 ROI 自动判断

```
每日计算各渠道 ROI：

ROI = GMV / 广告花费

自动行动建议：
- ROI > 2.0 → "建议加投，提高预算 {amount} 元"
- ROI 1.0-2.0 → "ROI 在观察区间，建议：{优化建议}"
- ROI < 1.0 → "⚠️ ROI 低于 1，建议暂停投放并分析原因：{诊断}"
```

#### 4.3 A/B 测试

根据数据表现生成多版本素材进行对比测试：

```
A/B 测试维度：
1. 视频封面（2-3 个变体）
2. 视频开头（不同 Hook 类型）
3. CTA 文案（不同行动引导）
4. 投放人群（不同年龄/地域/兴趣）

测试规则：
- 每次只测试一个变量
- 每组至少 1000 曝光量
- 数据差异 > 20% 时判定有显著差异
- 选择表现更好的版本继续投放
```

## 失败路径

### 情况 1: MVP 未达标 — 流量问题

```
诊断：视频播放量低，但点击→转化率正常
↓
行动：
1. 分析视频完播率 → 低于 20% → 优化视频开头（3 秒法则）
2. 分析发布时间 → 是否在高峰时段发布
3. 分析标题/封面 → 是否足够吸引点击
4. 增加发布频率 → 从每天 1 条增加到 2-3 条
↓
调整后重试 7 天
```

### 情况 2: MVP 未达标 — 转化问题

```
诊断：播放量正常，但点击→购买转化率低
↓
行动：
1. 分析商品页 → 价格/图片/描述是否有问题
2. 分析支付环节 → 支付流程是否顺畅
3. 分析用户评论 → 是否有阻碍购买的疑虑
4. 考虑降价测试 → 是否价格敏感
↓
调整后重试 7 天
```

### 情况 3: 投放 ROI 持续 < 1

```
诊断：广告花钱但不赚钱
↓
行动层级：
1. 优化素材 → 更换视频/封面/文案
2. 优化定向 → 缩窄人群/排除无效人群
3. 优化出价 → 调整 CPC/CPA 目标
4. 更换渠道 → 从抖音换到视频号投放（或反之）
5. 暂停投放 → 回归自然流量运营
↓
持续 2 周 ROI < 1 → 暂停投放，回归内容驱动
提示："付费投放暂时效果不佳，建议：
      1. 聚焦自然流量内容运营
      2. 提高内容质量和发布频率
      3. 等积累更多社交证明（评价/晒单）后再尝试投放
      4. 考虑合作 KOC/达人带货（以佣金模式降低风险）"
```

### 情况 4: 2 次 MVP 失败

```
暂停所有推进，进行根因分析
↓
根因诊断框架：
1. 产品本身的问题？→ 返回 Phase 2/3
2. 定价问题？→ 返回 Phase 5 调整
3. 目标人群不匹配？→ 返回 Phase 3
4. 品类选择失误？→ 返回 Phase 1
5. 执行问题（发布时间/平台选择）？→ 调整后重试

提示用户：
"MVP 两次未达标，在继续投入之前需要诚实地评估：
 是产品/市场的问题，还是执行的问题？

 建议选择一项行动：
 A. 深入分析：我来帮你做用户反馈回访分析
 B. 产品调整：返回 Phase 5 调整产品/定价
 C. 品类重选：返回 Phase 1 重新选择品类
 D. 暂停项目：记录经验教训，日后重启"
```

## 注意事项

1. MVP 验证是最关键的门槛，不要跳过直接投放——这是"烧钱"和"花钱"的区别
2. 存量激活策略是本框架的核心差异化——大多数教程都忽略了"先用免费流量验证"
3. 投放预算应该是渐进式的，不要一次性投入大量预算
4. 每日数据追踪是纪律，不是可选项——数据驱动是增长的基础
5. A/B 测试是提高 ROI 的核心手段，不要凭感觉决策
6. 退货率是隐藏的利润杀手，必须密切关注并分析退货原因
7. 如果用户不愿意每日追踪数据，建议降低到每 3 天一次，但不能完全不追踪
8. 增长是一个持续迭代的过程，不是一次性的事件——保持耐心和数据敏感度
