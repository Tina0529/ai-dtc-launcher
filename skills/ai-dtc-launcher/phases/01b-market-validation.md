# Phase 1b: 市场深度验证 (Market Validation)

## 阶段目标

对用户选定的品类（来自 Phase 1a 或用户直接指定）进行深度市场验证，从市场规模、竞争格局、供应链可行性和合规要求四个维度做全面评估，输出 Go/No-Go 建议。

---

## 输入

| 参数 | 来源 | 说明 |
|------|------|------|
| 选定品类 | Phase 1a 输出 / 用户指定 | 1-2 个待验证品类名称 |
| 蓝海评分卡 | Phase 1a 输出（如有） | 已有的评分数据，避免重复采集 |
| 目标市场 | meta.json | china / japan / global |

---

## 执行步骤

### Step 1: 市场规模和增长率分析

使用 WebSearch 搜索以下关键词，获取市场规模和增长数据：

```
搜索查询：
- "{品类名} 市场规模 2025 2026"
- "{品类名} 行业报告 市场增长率"
- "{品类名} market size growth rate"
- "{品类名} TAM SAM SOM"
```

**需要确认的数据点**：

| 指标 | 数据 | 来源 | 时效性 |
|------|------|------|--------|
| 全球市场规模 | {N} 亿元/美元 | {report_name} | {year} |
| 中国市场规模 | {N} 亿元 | {report_name} | {year} |
| 年复合增长率 (CAGR) | {N}% | {report_name} | {period} |
| 预计 2028 年市场规模 | {N} 亿元 | {report_name} | {year} |

**判定标准**：
- CAGR > 15% → 高速增长，强信号
- CAGR 10-15% → 稳健增长，正面信号
- CAGR 5-10% → 增长放缓，需谨慎
- CAGR < 5% → 接近饱和，负面信号

### Step 2: 头部玩家梳理

使用 WebSearch 搜索该品类的头部玩家和竞争格局：

```
搜索查询：
- "{品类名} Top 10 品牌 排名"
- "{品类名} 市场份额 头部品牌"
- "淘宝 {品类名} 销量排行"
- "抖音 {品类名} 头部卖家"
```

**输出格式**：

| 排名 | 品牌/店铺名 | 平台 | 月销量/GMV | 价格区间 | 品牌定位 | 核心卖点 |
|------|-----------|------|----------|---------|---------|---------|
| 1 | {name} | {platform} | {sales} | {price_range} | {positioning} | {usp} |
| 2 | ... | ... | ... | ... | ... | ... |
| ... | ... | ... | ... | ... | ... | ... |

**分析要点**：
- Top 3 市占率是否超过 50%？（超过则头部集中度高）
- 是否有知名品牌？（有则品牌密度高）
- 价格区间分布是否有空白带？（有则存在定价机会）
- 新品牌（成立 < 2 年）的占比？（高则说明市场仍在开放）

### Step 3: 供应链可行性

使用 `ecommerce-searcher` 在 1688 搜索该品类产品：

```
搜索参数：
- 关键词: {品类名} / {产品关键词}
- 平台: 1688
- 排序: 综合排序
```

**需要采集的数据**：

| 指标 | 数据 | 说明 |
|------|------|------|
| 供应商数量 | {N} 家 | 搜索结果页面总数 |
| 价格区间 | {min} - {max} 元 | 前 20 个结果的价格范围 |
| 平均 MOQ | {N} 件 | 最低起订量的中位数 |
| 支持定制 | {是/否} | 是否支持 LOGO/包装定制 |
| 发货地集中度 | {地区} | 主要产地/发货地 |
| 一件代发 | {是/否} | 是否支持一件代发 |

**供应链可行性判定**：

| 条件 | 判定 | 说明 |
|------|------|------|
| 供应商 > 100 家，MOQ ≤ 100 件 | 绿灯 | 供应链成熟，个人可操作 |
| 供应商 30-100 家，MOQ 100-500 件 | 黄灯 | 可行但需更多启动资金 |
| 供应商 < 30 家，MOQ > 500 件 | 红灯 | 供应链不成熟或门槛高 |

**日本市场补充**（如适用）：
- 使用 `amazon-japan-shopping` 搜索日本市场同类产品
- 确认是否需要额外的物流/清关安排
- 调研日本市场的价格水平（通常高于中国 2-5 倍）

### Step 4: 资质合规初判

使用 WebSearch 搜索该品类的合规要求：

```
搜索查询：
- "{品类名} 需要什么证照 资质"
- "{品类名} 电商销售 合规要求"
- "{品类名} 营业执照 经营范围"
- "抖音小店 {品类名} 入驻资质"
- "视频号小店 {品类名} 类目要求"
```

**输出格式**：

| 资质项目 | 是否必需 | 获取难度 | 预估费用 | 预估时间 | 备注 |
|---------|---------|---------|---------|---------|------|
| 营业执照 | 必需 | 低 | 0-500元 | 3-5天 | 个体工商户即可 |
| {特殊资质1} | {必需/推荐/不需要} | {高/中/低} | {fee} | {time} | {note} |
| {特殊资质2} | ... | ... | ... | ... | ... |

**合规风险判定**：

| 条件 | 判定 | 说明 |
|------|------|------|
| 仅需营业执照，无特殊资质 | 绿灯 | 合规门槛低 |
| 需要 1-2 项可获取的特殊资质 | 黄灯 | 需要额外时间和费用，但可操作 |
| 需要多项高门槛资质（医疗器械/药品等） | 红灯 | 合规门槛过高，不适合个人创业 |

## 检查点

**暂停执行，展示验证报告给用户，等待用户确认。**

提示用户：
```
以上是 "{品类名}" 的市场深度验证报告。

综合建议：{GO / CONDITIONAL GO / NO-GO}

请确认：
1. ✅ 确认进入此市场 → 进入 Phase 2 选品雷达
2. 🔄 需要补充某些数据 → 我来补充分析
3. ❌ 放弃此品类 → 返回 Phase 1a 选择下一个候选品类
```

**用户确认后**：
- GO → 更新 `meta.json`，`current_phase` 设为 `phase-2`，保存报告到 `market-report.md`
- 返回 → 更新 `meta.json`，`current_phase` 回退到 `phase-1a`

## 失败路径

### 情况 1: No-Go

```
行动：返回 Phase 1a 的排行榜，选择下一个候选品类
↓
下一个候选品类进入 Phase 1b 验证
↓
所有候选品类都 No-Go → 建议用户重新扫描（放宽约束或换市场）
```

### 情况 2: 数据不足无法判定

```
行动：列出缺失数据项，请用户协助提供
提示："以下数据对做出准确判断至关重要，但我无法自动获取：
      1. {缺失数据1} — 建议从 {数据源} 获取
      2. {缺失数据2} — 建议从 {数据源} 获取
      请提供数据后我来完成验证，或选择基于现有数据做保守判断。"
```

### 情况 3: Conditional Go 但条件难以满足

```
行动：与用户讨论条件的可行性
  - 如果条件可在 1-2 周内满足 → 建议并行推进
  - 如果条件需要 > 1 个月 → 建议先推进其他品类
  - 如果条件需要大量额外资金 → 重新评估是否值得投入
```

---

## 注意事项

1. 验证一个品类的完整流程预计需要 20-30 分钟
2. 如果用户选了 2 个品类，串行执行（先验证第一个，再验证第二个）
3. 如果 Phase 1a 已有趋势验证数据，直接复用，不要重复搜索
4. 市场规模数据优先使用权威报告（艾瑞、Statista、尼尔森等），博客/自媒体数据仅作参考
5. 供应链验证重点关注 MOQ 和是否支持定制，这两个是个人创业者的关键约束
