# Phase 4: 品牌建筑师 (Brand Architect)

## 阶段目标

基于 Phase 3 的消费者洞察和差异化方向，定义完整的品牌体系：目标用户画像、品牌调性、视觉体系和品牌命名方案。输出品牌简报（Brand Brief），作为后续产品设计和营销的指导纲领。

---

## 输入

| 参数 | 来源 | 说明 |
|------|------|------|
| 需求图谱 | Phase 3 输出 | 消费者核心需求和痛点 |
| 品牌真空报告 | Phase 3 输出 | 品牌提及率和心智地图 |
| 差异化方向 | Phase 3 检查点 | 用户确认的切入角度 |
| 产品信息 | Phase 2 输出 | 产品名/价格带/竞品 |
| 目标市场 | meta.json | china / japan / global |

---

## 执行步骤

### Step 1: 用户画像生成

#### 1.1 评论数据分析

从 Phase 3 采集的用户评论中提取人群特征信号：

```
AI 分析 Prompt：
"请从以下用户评论中推断目标消费者的人群画像。

分析维度：
1. 年龄段推断（从用语习惯、关注点推断）
2. 性别倾向（从产品偏好、表达方式推断）
3. 消费能力（从价格敏感度、对比品牌推断）
4. 生活方式（从使用场景、购买动机推断）
5. 信息渠道（从发现产品的方式推断）

用户评论数据：{Phase 3 的评论数据摘要}
需求图谱数据：{Phase 3 的需求图谱}
"
```

#### 1.2 核心画像标签提取

从分析结果中提炼 3-5 个核心画像标签。每个标签需要有数据支撑。

**标签生成规则**：
- 标签应是可操作的（能指导设计和营销决策）
- 标签应是差异化的（区别于泛泛的"年轻人"、"女性"等）
- 标签应有情感维度（不仅是人口统计，还要有态度/价值观）

示例标签：
- "精致懒人" — 追求品质但不愿花太多时间挑选
- "理性种草" — 不盲目跟风，会看差评和对比
- "颜值正义" — 愿意为好看的设计付溢价

#### 1.3 审美标签匹配

**操作**：加载 `engines/persona-brand-matcher.md` 的审美标签库进行匹配。

> 如果 `engines/persona-brand-matcher.md` 尚未创建，使用以下简化匹配：

根据画像标签匹配审美偏好：

| 画像标签 | 审美偏好 | 品牌调性方向 | 视觉风格方向 |
|---------|---------|------------|------------|
| {tag_1} | {aesthetic_1} | {tone_1} | {visual_1} |
| {tag_2} | {aesthetic_2} | {tone_2} | {visual_2} |
| ... | ... | ... | ... |

### Step 2: 品牌调性对标

#### 2.1 参考品牌识别

AI 识别 3 个调性相近的参考品牌（不限于同品类）：

```
AI 分析 Prompt：
"基于以下目标人群画像和差异化方向，推荐 3 个调性相近的参考品牌。

要求：
1. 不一定是同品类，但调性/受众/价值观相近
2. 至少 1 个是 DTC 原生品牌（有线上营销经验可借鉴）
3. 至少 1 个是视觉设计出色的品牌
4. 优先选择用户画像重合度高的品牌

目标人群画像：{persona_description}
核心标签：{tags}
差异化方向：{differentiation}
产品品类：{category}
"
```

#### 2.2 视觉 DNA 分析

对每个参考品牌进行视觉 DNA 分析：

使用 WebSearch 搜索每个品牌的视觉资料：
```
搜索查询：
- "{brand_name} 品牌设计 视觉"
- "{brand_name} 品牌定位 调性"
- "{brand_name} 官网 产品图"
- "{brand_name} 小红书 包装"
```

**分析维度**：

| 维度 | 品牌 1 | 品牌 2 | 品牌 3 |
|------|--------|--------|--------|
| 主色调 | {描述} | {描述} | {描述} |
| 排版风格 | {描述} | {描述} | {描述} |
| 摄影风格 | {描述} | {描述} | {描述} |
| 包装设计 | {描述} | {描述} | {描述} |
| 品牌故事 | {描述} | {描述} | {描述} |
| 社交媒体风格 | {描述} | {描述} | {描述} |

### Step 3: AI 视觉方案生成

#### 3.1 Mood Board 生成

调用 `gemini-imagegen` 生成 3 套 Mood Board：

**生成 Prompt 模板**：
```
方案 A（{风格关键词，如：温暖自然}）：
"Create a mood board for a {product_category} brand targeting {persona_description}.
Style: {style_keywords}.
Color palette: {color_direction}.
Include product photography style, packaging concept, typography suggestion, and lifestyle imagery.
Brand personality: {brand_personality}."

方案 B（{风格关键词}）：
{类似结构，不同风格方向}

方案 C（{风格关键词}）：
{类似结构，不同风格方向}
```

#### 3.2 配色方案提取

为每个 Mood Board 方案定义具体配色：

| 色彩角色 | 方案 A | 方案 B | 方案 C |
|---------|--------|--------|--------|
| 主色 | #{hex} {色名} | #{hex} {色名} | #{hex} {色名} |
| 辅色 | #{hex} {色名} | #{hex} {色名} | #{hex} {色名} |
| 强调色 | #{hex} {色名} | #{hex} {色名} | #{hex} {色名} |

#### 3.3 平台视觉适配评估

调用 `content-scoring` 评估每套方案的平台适配度：

```
评估维度：
- 小红书视觉适配度（高颜值/种草感）
- 抖音视频封面适配度（醒目/信息密度）
- 微信视频号适配度（信任感/品质感）
- 电商主图适配度（清晰/卖点突出）
```

---

### Step 4: 品牌命名

#### 4.1 AI 候选名生成

```
AI 生成 Prompt：
"请为以下品牌生成 20 个候选品牌名。

品牌信息：
- 品类：{category}
- 目标人群：{persona}
- 品牌调性：{tone_keywords}
- 差异化方向：{differentiation}
- 目标市场：{market}

命名要求：
1. 2-4 个字/音节，易记易搜
2. 无负面谐音（中文/日文/英文均需检查）
3. 暗示品类或品牌调性（但不要太直白）
4. 朗朗上口，有品牌感（不像山寨）
5. 建议混合风格：纯中文/英文/中英混合/创造词

输出格式：
1. {品牌名} — {含义/寓意} — {命名风格：纯中文/英文/创造词/...}
2. ...（共 20 个）
"
```

#### 4.2 商标可用性检查

对 Top 10 候选品牌名，使用 WebSearch 检查商标注册状态：

```
搜索查询（每个品牌名）：
- "{品牌名} 商标注册"
- "{品牌名} 商标 第{N}类"（N = 该品类对应的商标分类）
- "中国商标网 {品牌名}"
```

**判定标准**：
- 未找到相同/近似注册 → 可注册
- 有近似但不同类别 → 需进一步确认
- 已有相同类别注册 → 不可用

#### 4.3 域名可用性检查

使用 WebSearch 检查域名可用性：

```
搜索查询（每个品牌名）：
- "{品牌名拼音}.com 域名"
- "{品牌名拼音}.cn 域名"
- "{brand_name_en}.com 域名查询"
```

#### 4.4 社交账号可用性检查

使用 WebSearch 检查各平台账号名可用性：

```
搜索查询（每个品牌名）：
- "微信公众号 {品牌名}"
- "抖音 {品牌名}"
- "小红书 {品牌名}"
- "微博 {品牌名}"
```

#### 4.5 候选名排序

综合商标/域名/账号可用性，筛选 Top 3-5 个推荐品牌名。

## 输出

### 品牌简报

使用 `templates/brand-brief.md` 模板，填写完整的品牌简报。

包含：
1. 目标人群画像（一段话描述 + 3-5 个核心标签）
2. 品牌调性关键词（3-5 个，含释义和视觉方向）
3. 参考品牌（3 个，含视觉 DNA 分析）
4. 配色方案（推荐方案的完整色值）
5. 字体建议
6. 品牌名候选（3-5 个，含商标/域名/账号状态）
7. Mood Board 描述

保存到 `brand-brief.md`。

---

## 检查点

**暂停执行，展示品牌简报给用户，等待用户确认。**

提示用户：
```
以上是 "{品牌名候选}" 的品牌简报。

需要您确认以下关键决策：

1. **品牌名**：推荐 {top_name}，您觉得如何？
   备选：{alternatives}

2. **视觉风格**：推荐方案 {A/B/C}，理由：{reason}
   您更喜欢哪个方向？

3. **品牌调性**：{tone_keywords}
   是否需要调整？

请确认后进入 Phase 5 产品落地。

您也可以：
- 要求重新生成品牌名
- 要求调整视觉方案
- 混搭不同方案的元素
```

**用户确认后**：
- 更新 `meta.json`：记录品牌名和视觉方案，`current_phase` 设为 `phase-5`
- 保存确认后的品牌简报为最终版

## 失败路径

### 情况 1: 用户对所有品牌名都不满意

```
行动：收集用户偏好后重新生成
提示："没问题！能告诉我：
      1. 有没有喜欢的品牌名风格？（日系简约/国潮文艺/英文现代/...）
      2. 有没有想包含的字/音？
      3. 有没有参考的品牌名（其他品类的也行）？"
→ 基于反馈重新生成 20 个候选名
→ 最多重试 3 轮
```

### 情况 2: 所有候选品牌名的商标都已注册

```
行动：
1. 尝试创造词（将两个现有词组合/变形）
2. 尝试加前缀/后缀（如"小X"、"X研究所"）
3. 使用纯英文或造词（如"Muji"模式）
4. 建议用户咨询商标代理确认可注册范围
```

### 情况 3: 用户有强烈的个人审美偏好

```
行动：以用户偏好为主，AI 提供专业建议作为参考
- 如果用户偏好与目标人群审美有较大偏差 → 温和提醒
  "您的偏好是 {user_pref}，但目标消费者更倾向 {data_pref}。
   建议考虑在两者之间找到平衡，或者做 A/B 测试验证。"
- 不要强行推翻用户的审美决策
```

---

## 注意事项

1. 品牌简报是整个项目的核心纲领，后续所有视觉产出都应参考此文档
2. 品牌命名是最关键的决策之一，建议给用户充分的思考时间
3. 商标检查只是初步筛查，正式注册前需要专业商标代理的查询
4. Mood Board 生成可能需要多次迭代，不要急于确定
5. 配色方案需要考虑在不同平台（白底/暗色背景）的表现
6. 字体选择需要考虑授权费用，创业初期优先使用免费可商用字体
7. 如果目标市场包含日本，品牌名需要额外检查日语发音和联想
