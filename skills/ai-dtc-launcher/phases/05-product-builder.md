# Phase 5: 产品落地 (Product Builder)

## 阶段目标

将品牌概念转化为可销售的实体产品。完成产品设计、供应链对接、销售渠道搭建、电商素材制作和合规检查，所有准备工作就绪后通过 Launch Checklist 验收。

---

## 输入

| 参数 | 来源 | 说明 |
|------|------|------|
| 品牌简报 | Phase 4 输出 | 品牌名/视觉体系/调性/配色 |
| 产品信息 | Phase 2 输出 | 产品名/采购价/竞品售价 |
| 消费者洞察 | Phase 3 输出 | 痛点/需求/差异化方向 |
| 供应链数据 | Phase 1b/2 输出 | 供应商/MOQ/价格 |
| 目标市场 | meta.json | china / japan / global |

---

## 执行步骤

### Step 1: AI 产品设计

#### 1.1 产品款式/配色生成

基于 Phase 4 品牌视觉方案，确定产品的具体款式和配色方案。

```
AI 分析 Prompt：
"基于以下品牌视觉方案和目标消费者偏好，为 '{product_name}' 设计 3-5 款产品变体。

品牌配色方案：
- 主色：{primary_color}
- 辅色：{secondary_color}
- 强调色：{accent_color}

目标消费者偏好：{Phase 3 需求图谱中的正面需求}
差异化方向：{differentiation}
竞品常见款式：{competitor_styles}

请生成：
1. 每款变体的颜色/材质/细节设计说明
2. 设计理念（与品牌调性的关联）
3. 预估成本差异
4. 建议首批 SKU 数量（控制在 3-5 个）
"
```

#### 1.2 产品效果图生成

调用 `gemini-imagegen` 生成产品效果图：

```
生成 Prompt（每款变体各 1 张）：
"Professional product photography of {product_description},
 in {color_description},
 style: {brand_visual_style},
 on white background with subtle shadow,
 studio lighting, 4K quality, commercial product shot.
 Brand color palette: {color_palette}."
```

**建议生成数量**：3-5 款变体 × 每款 1 张 = 3-5 张效果图

#### 1.3 产品方案确认

将效果图和设计说明展示给用户，确认最终产品方案和首批 SKU。

### Step 2: 供应链对接

#### 2.1 供应商筛选

使用 `ecommerce-searcher` 在 1688 深度搜索供应商：

```
搜索策略：
1. 常规搜索："{product_name}"
2. 源头工厂搜索："{product_name} 源头工厂"
3. 定制搜索："{product_name} 定制 LOGO"
4. OEM 搜索："{product_name} OEM 加工"
```

**供应商筛选维度**：

| 维度 | 优选条件 | 权重 |
|------|---------|------|
| 信用等级 | 实力商家/超级工厂 | 20% |
| 起订量 (MOQ) | ≤ 100 件 | 20% |
| 打样能力 | 支持打样 | 15% |
| 定制能力 | 支持 LOGO/包装定制 | 15% |
| 价格竞争力 | 价格处于中位或偏低 | 15% |
| 发货速度 | 48 小时内发货 | 10% |
| 交易量 | 近 90 天成交笔数 > 50 | 5% |

**输出：供应商短名单**

| 排名 | 供应商名 | 信用等级 | 单价 | MOQ | 打样费 | 定制 | 发货地 | 推荐度 |
|------|---------|---------|------|-----|--------|------|--------|--------|
| 1 | {name} | {level} | {price} | {moq} | {fee} | {是/否} | {location} | {推荐/备选} |
| 2 | ... | ... | ... | ... | ... | ... | ... | ... |
| 3 | ... | ... | ... | ... | ... | ... | ... | ... |

#### 2.2 询价话术生成

为用户生成标准化询价话术模板：

```markdown
## 询价话术模板

---

您好！我是 {品牌名} 的创始人，正在寻找 {产品名} 的合作供应商。

我们是新品牌，计划首批小量试单，后续稳定返单。想了解以下信息：

1. **样品**：能否先寄样品？打样费用是多少？可以抵扣货款吗？
2. **起订量**：最低起订量是多少件？首单 {50-100} 件的单价是？
3. **定制**：是否支持以下定制？
   - LOGO 印制/刺绣
   - 包装盒定制
   - {其他定制需求}
4. **交货周期**：下单到发货需要多少天？
5. **付款方式**：支持什么付款方式？（30% 定金 + 70% 发货前？）
6. **退换货**：如果有质量问题，退换货政策是什么？
7. **可否提供质检报告？**

期待合作！

{联系方式}

---
```

#### 2.3 成本结构计算

**操作**：加载 `templates/cost-structure.md`，填写完整的成本结构。

**数据来源**：
- 采购成本：1688 询价数据
- 物流费用：快递公司报价 / 行业均价估算
- 包装费用：1688 包装供应商报价
- 平台费率：各平台官方公布费率
- 退货率：品类行业平均值（Phase 1b 数据）

#### 2.4 首单策略

**核心原则：风险控制小批量**

```
首单建议：
- 数量：50-100 件（不要贪多）
- 理由：验证市场需求，降低库存风险
- SKU：首批控制 2-3 个 SKU（主打款 + 1-2 个变体）
- 总投入：{estimated_total}（采购+物流+包装）
- 最坏情况：全部滞销损失 {max_loss} 元

风险控制规则：
- 首单总投入 ≤ 用户预算的 30%
- 单 SKU 数量 ≤ 50 件
- 预留 50% 资金用于补货和营销
```

**检查点：用户确认供应商和首单量**

```
提示用户：
"建议联系以上 Top 3 供应商询价和要样品。

首单建议：
- 数量：{quantity} 件
- 预估总投入：{total} 元
- SKU：{sku_plan}

请确认：
1. ✅ 同意首单计划 → 开始联系供应商（用户操作）
2. 🔄 调整数量/SKU → 我来重新计算成本
3. ❓ 需要更多供应商选择 → 我来继续搜索"
```

### Step 3: 销售渠道搭建

#### 3.1 首选渠道：微信小商店

**选择理由**：利用现有视频号 3.8 万粉丝，无需额外获客成本。

**搭建指南**：

```markdown
## 微信小商店搭建步骤

1. **开通小商店**
   - 微信搜索"小商店助手"
   - 选择"个人/企业"类型
   - 完成实名认证

2. **关联视频号**
   - 进入小商店后台 → 渠道管理 → 视频号
   - 绑定已有视频号（3.8万粉丝）

3. **商品上架**
   - 添加商品 → 填写标题/描述/价格/库存
   - 上传主图和详情图（Step 4 生成）
   - 设置运费模板

4. **支付设置**
   - 开通微信支付（个人使用个人收款码，企业使用商户号）
   - 设置结算银行账户

5. **售后规则**
   - 设置 7 天无理由退货
   - 配置退货地址
   - 设置退款自动处理规则
```

#### 3.2 备选渠道

**抖音小店**（需营业执照）：
```
搭建要点：
- 注册抖音小店（需个体工商户营业执照）
- 缴纳保证金（品类不同，通常 2000-5000 元）
- 商品入驻审核（1-3 个工作日）
- 开通精选联盟（达人带货渠道）
```

**拼多多**（需营业执照）：
```
搭建要点：
- 注册拼多多商家（个人店/企业店）
- 缴纳保证金（品类不同，1000-10000 元）
- 上架商品（注意拼多多的标题SEO规则）
```

**日本市场渠道**（如适用）：

| 渠道 | 门槛 | 费用 | 适合阶段 |
|------|------|------|---------|
| Amazon JP | Seller Central 注册 | 月额 ¥4,900 + 手数料 8-15% | MVP 验证 |
| 楽天市場 | 出店審査 | 月額 ¥19,500~ + 手数料 2-7% | 品牌建设期 |
| BASE | 简易注册 | 手数料 6.6% + 40円/件 | 冷启动测试 |

### Step 4: 电商素材包

#### 4.1 主图生成

调用 `gemini-imagegen` 生成电商主图：

```
白底主图 Prompt：
"Professional product photo of {product_description} on pure white background,
 studio lighting, centered composition, 800x800px,
 e-commerce product listing style,
 brand color: {primary_color}."

场景图 Prompt（2-3张）：
"Lifestyle product photography of {product_description} in {usage_scene},
 natural lighting, {brand_mood} atmosphere,
 target audience: {persona_description},
 social media ready, high quality."
```

**主图套装**：
1. 白底正面图 × 1
2. 使用场景图 × 2-3
3. 细节/材质特写 × 1-2
4. 尺寸/规格图 × 1（如适用）

#### 4.2 详情页文案

从 Phase 3 的痛点到卖点的映射：

```
AI 生成 Prompt：
"请为 '{product_name}' 生成电商详情页文案。

核心策略：将消费者痛点转化为产品卖点。

痛点 → 卖点映射：
{pain_point_1} → {selling_point_1}
{pain_point_2} → {selling_point_2}
{pain_point_3} → {selling_point_3}

文案结构：
1. 开头 Hook（针对最大痛点）
2. 3-5 个核心卖点（图文结合）
3. 使用场景展示
4. 品质/材质说明
5. 尺寸/规格参数
6. 品牌故事（简短）
7. 售后保障说明

品牌调性：{brand_tone}
目标平台：{platform}
"
```

#### 4.3 素材质量评估

调用 `content-scoring` 评估整套素材的质量：

```
评估对象：
- 主图组（5-7 张）
- 详情页文案
- 产品标题

评估标准：
- 视觉吸引力（主图是否在搜索结果中脱颖而出）
- 信息完整度（消费者关心的信息是否都覆盖了）
- 卖点清晰度（核心卖点是否在前 3 屏就能看到）
- 品牌一致性（配色/字体/调性是否统一）

目标分数：≥ 80 分
```

**如果评分 < 80 分**：
1. 查看具体扣分项
2. 针对性优化（文案调整/图片重新生成/排版优化）
3. 重新评分直到 ≥ 80 分

## 失败路径

### 情况 1: 样品质量不达标

```
行动：
1. 记录不达标的具体问题
2. 联系供应商反馈 → 要求改进后重新打样
3. 如果供应商无法改进 → 切换到短名单中的备选供应商
4. 如果所有供应商都无法达标 → 返回 Phase 2，考虑换产品

提示用户：
"样品质量是底线，不要为了赶时间而将就。
 消费者体验差 → 差评 → 退货率高 → 品牌受损
 多花 1-2 周找到合适的供应商，远比后续救火值得。"
```

### 情况 2: 合规资质办理困难

```
行动：
1. 评估是否真的必需（有些资质是推荐而非必需）
2. 是否有代办服务可以加速
3. 是否可以先在不需要该资质的渠道销售（如微信小商店 vs 天猫）
4. 如果资质门槛过高 → 返回 Phase 1b，重新评估品类
```

### 情况 3: content-scoring 评分持续 < 80 分

```
行动：
1. 分析最大扣分项
2. 如果是图片质量 → 重新生成或建议用户拍摄实物照
3. 如果是文案 → 调整文案结构和卖点表达
4. 如果是信息缺失 → 补充规格/材质/售后信息
5. 最多优化 3 轮，如果仍不达标 → 降低到 75 分可接受（标注需后续优化）
```

---

## 注意事项

1. 产品效果图≠实物，一定要等到样品到手后拍实物照替换
2. 成本结构中的预估值需要在实际采购后校准
3. 首单坚持小批量原则，不要被供应商"量大优惠"诱惑
4. 电商素材需要根据不同平台要求调整（抖音更强调视频，小红书更强调图文）
5. 合规是底线，不要心存侥幸——一次违规可能导致店铺被封
6. 从下单到产品就绪，通常需要 2-4 周，这段时间可以并行准备内容和渠道
